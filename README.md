## 1. Калибровка камеры

Калибровка необходима для определения оптических параметров вашей камеры (матрицы камеры и коэффициентов дисторсии), чтобы точно определять 3D-положение ArUco-маркеров.

### Метод и подготовка
1. Распечатайте калибровочную шахматную доску (например, с сеткой $9 \times 7$ квадратов, что даёт $8 \times 6$ внутренних углов).
2. Замерьте размер стороны квадрата линейкой в метрах (например, `0.025` м для квадрата 25 мм).
3. Запишите видео на камеру робота, плавно поворачивая шахматную доску под разными углами и расстояниями.
4. Положите видеофайл в проект (например, по пути `src/fake_tag_publisher/config/calibration.mp4`).

### Команда запуска калибровки
Находясь в папке воркспейса `/home/panik_bel/ros2_ws`, запустите специальную ноду:

```bash
ros2 run fake_tag_publisher camera_calibrator --ros-args \
  -p video_path:=src/fake_tag_publisher/config/calibration.mp4 \
  -p square_size:=0.025 \
  -p board_width:=8 \
  -p board_height:=6 \
  -p frame_step:=15
```

**Параметры:**
* `video_path` — путь к файлу калибровки.
* `square_size` — физический размер стороны квадрата в метрах.
* `board_width` — количество внутренних пересечений (углов) по горизонтали (на единицу меньше количества квадратов).
* `board_height` — количество внутренних пересечений (углов) по вертикали.
* `frame_step` — с каким шагом брать кадры из видео (например, каждый 15-й кадр), чтобы ускорить процесс и получить разнообразные ракурсы.

**Результат:**
Программа автоматически найдет углы на видеокадрах, выполнит расчеты OpenCV и автоматически сохранит результаты в файл конфигурации камеры [camera_info.yaml](file:///home/panik_bel/ros2_ws/src/fake_tag_publisher/config/camera_info.yaml).

## 2. Управление ArUco метками

База данных меток находится в файле конфигурации [tags_config.yaml](file:///home/panik_bel/ros2_ws/src/fake_tag_publisher/config/tags_config.yaml).

### Какое семейство меток использовать?
По умолчанию в проекте настроено семейство **DICT_4X4_100** (или **DICT_4X4_50**). 
* Маркеры этого семейства имеют размерность $4 \times 4$ точки, что делает их простыми для распознавания на расстоянии и при слабом освещении.
* Вы можете генерировать и скачивать маркеры этого семейства с помощью бесплатных онлайн-генераторов ArUco (например, [Aruco Generator](https://chev.me/arucogen/)). Укажите словарь `Original ArUco` или `4x4`, введите нужный ID метки (например, `67`).

### Как изменить тип словаря в коде/запуске?
Вы можете передать тип словаря при запуске через аргумент `aruco_dictionary`. Поддерживаются стандартные типы OpenCV (например, `DICT_4X4_100`, `DICT_5X5_250`, `DICT_6X6_250` и т.д.):
```bash
ros2 launch fake_tag_publisher run_localization_video.launch.py aruco_dictionary:=DICT_6X6_250
```

### Как добавить или удалить метку?
Откройте файл [tags_config.yaml](file:///home/panik_bel/ros2_ws/src/fake_tag_publisher/config/tags_config.yaml):
* **Чтобы добавить:** Вставьте новую секцию под `tags:`, указав её ID и точные координаты на карте (положение на потолке):
  ```yaml
    tag_67:
      x: 0.0          # Координата X на карте (в метрах)
      y: 0.0          # Координата Y на карте (в метрах)
      z: 2.5          # Высота потолка (в метрах)
      roll: 3.1415926 # Разворот вниз лицом (180 градусов в радианах)
      pitch: 0.0
      yaw: 0.0        # Угол рыскания (ориентация метки относительно оси X карты)
  ```
* **Чтобы удалить:** Просто сотрите или закомментируйте (символом `#`) соответствующий блок `tag_ID:`.

## 3. Запуск локализации с новыми видеороликами

Благодаря гибкой настройке путей, вы можете запускать ноду с любыми тестовыми видеороликами.

1. Положите новое видео в папку `src/fake_tag_publisher/config/` (например, `new_drive.mp4`).
2. Запустите лонч-файл с указанием этого видео, флагом циклического воспроизведения и коэффициентом фильтрации:
   ```bash
   ros2 launch fake_tag_publisher run_localization_video.launch.py \
     video_path:=src/fake_tag_publisher/config/new_drive.mp4 \
     loop_video:=False \
     filter_alpha:=0.15
   ```
   * `loop_video:=False` — видео проиграется один раз, после чего нода сохранит статистику, построит график и корректно завершит работу.
   * `filter_alpha:=0.15` — коэффициент сглаживания фильтра (чем меньше число, тем сильнее сглаживание шумов детекции, но больше задержка; рекомендуемый диапазон: `0.1` - `0.3`).

## 4. Анализ графиков траектории

Когда вы запускаете видео с параметром `loop_video:=False`, в конце воспроизведения нода локализации автоматически производит математические расчеты пути:
1. Вычисляет общую длину пути по сырым (зашумленным) координатам.
2. Вычисляет общую длину пути после фильтрации низких частот (EMA + Slerp), убирая sub-centimeter "дребезг" маркеров.
3. Строит сравнительный график траекторий.

### Где посмотреть график?
График сохраняется автоматически в двух местах:
1. В папке исходного кода: [trajectory_plot.png](file:///home/panik_bel/ros2_ws/src/fake_tag_publisher/config/trajectory_plot.png)
2. В папке сборки (share): `install/fake_tag_publisher/share/fake_tag_publisher/config/trajectory_plot.png`

Вы можете открыть его прямо из файлового менеджера или в VS Code для визуальной оценки сглаживания траектории движения вашего робота.
